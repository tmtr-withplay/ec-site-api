1 準備フェーズ（開発環境）

1.1 必須ソフトウェア（Windows 11 開発機）

  • Java：Amazon Corretto 17（開発・本番で同一バージョンを使用）
  • Build：Maven（プロジェクト管理・ビルド）
  • IDE：IntelliJ IDEA（Community で十分）
  • DB クライアント：pgAdmin / DBeaver（PostgreSQL 接続用）
  • Git：Git for Windows
  • VirtualBox：Ubuntu Server を動かすため

1.1.1 Amazon Corretto の導入（Windows）

理由：OpenJDK 互換で商用利用に安心、長期サポートがあるため本番と開発で同一環境を揃えやすい。

# ダウンロード・インストール（例）
1. https://aws.amazon.com/corretto/ から MSI をダウンロード
2. インストーラを実行して PATH に追加
3. 動作確認:
> java -version

1.1.2 Maven と IntelliJ の設定

IntelliJ は Maven プロジェクトを直接インポートできます。JDK を Corretto に設定してください。

1.2 Ubuntu Server（VirtualBox）準備

  1. VirtualBox に Ubuntu Server をインストール（最小構成、GUIなし）。
  2. SSH を有効化し、ホストとポートフォワード（例：ホスト 2222 → ゲスト 22）を設定。
  3. Ubuntu に Corretto、PostgreSQL、Nginx（リバースプロキシ）をインストール。
# Ubuntu 側の基本コマンド（例）
sudo apt update && sudo apt upgrade -y
# Corretto (例: 17)
sudo apt install -y wget
wget https://corretto.aws/downloads/latest/amazon-corretto-17-x64-linux-jdk.deb
sudo dpkg -i amazon-corretto-17-*-linux-x64.deb
java -version

# PostgreSQL
sudo apt install -y postgresql postgresql-contrib

# Nginx (リバースプロキシ)
sudo apt install -y nginx
注意：本番と同じ Java / PostgreSQL バージョンを開発環境にも揃えること（差異がバグの原因になります）。

2 設計フェーズ（必須設計項目と解説）

2.1 API 設計（RESTful）

設計方針（要点）：

  • リソース指向：動詞を使わない（例：/products, /orders）。
  • HTTP メソッドの適切な利用：GET/POST/PATCH/DELETE/PUT。
  • 認証不要は最小限（要件では商品一覧のみ認証不要）。
  • ページネーション：limit/offset（クエリパラメータ）。
  • エラーフォーマット統一：{ "error": "...", "message": "..." }。

代表的なエンドポイント（抜粋）

メソッドパス説明
POST/api/v1/auth/register会員登録
POST/api/v1/auth/loginログイン（JWT 発行）
GET/api/v1/products商品一覧（認証不要）
POST/api/v1/orders注文確定（トランザクション内で在庫更新・決済）

2.2 DB 設計（PostgreSQL）

依頼書の ER をベースに、正規化・インデックス・制約を設計します。主なポイント：

  • users：email に UNIQUE、password はハッシュ（bcrypt）
  • products：deleted_at による論理削除、price は DECIMAL
  • stocks：product_id に UNIQUE、在庫は CHECK (quantity >= 0)
  • orders / order_items：注文確定時に在庫を減算、order_items に unit_price を保持
  • インデックス：products(category_id), products(price), orders(user_id, created_at)

マイグレーション

Flyway または Liquibase を導入してマイグレーション管理を行うことを推奨。

3 実装フェーズ（Spring Boot + Maven）

3.1 プロジェクト構成（推奨）

com.example.ec
├─ Application.java
├─ config
│  ├─ SecurityConfig.java   # JWT, password encoder
│  └─ SwaggerConfig.java
├─ controller
├─ service
├─ repository
├─ dto
├─ entity
└─ exception

3.2 主要技術ポイント（実装上の注意）

  • 認証：JWT（アクセストークンに有効期限を設定）。ログアウトはトークン無効化（ブラックリスト or token version）。
  • パスワード：BCryptPasswordEncoder を使用してハッシュ化。
  • トランザクション：注文確定処理は @Transactional（REQUIRED）で在庫減算・注文作成・決済呼び出しを同一トランザクションで実行。
  • 決済モック：外部 API を想定したサービスを作成し、成功/残高不足/エラー の3パターンを返す。決済失敗時は例外を投げてロールバック。
  • 例外処理：@ControllerAdvice で統一エラーフォーマットを返す。
  • DB アクセス：Spring Data JPA（プレースホルダ/ORM）で SQL インジェクション対策。
  • パフォーマンス：商品一覧は必要なカラムのみ SELECT、カテゴリ・価格にインデックス、N+1 を防ぐため fetch join を活用。

3.3 ユニットテスト

必須：認証・注文確定（正常/在庫不足/決済失敗）・在庫更新のテスト。JUnit + Mockito を使用。

// 例: 注文確定のテスト（擬似コード）
@Test

  // arrange: モック在庫 10, カート 2 個

4 CI/CD（GitHub Actions → Ubuntu Server）

4.1 方針

GitHub Actions でビルド・テスト・パッケージング（jar）を実行し、成功したら Ubuntu Server（VirtualBox ゲスト）へ SSH でデプロイ。デプロイ後は systemd サービスでアプリを起動。

4.2 GitHub Actions ワークフロー（例）

name: CI-CD

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  build-and-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Set up JDK 17
        uses: actions/setup-java@v4
        with:
          distribution: 'corretto'
          java-version: '17'
      - name: Cache Maven packages
        uses: actions/cache@v4
        with:
          path: ~/.m2/repository
          key: ${{ runner.os }}-m2-${{ hashFiles('**/pom.xml') }}
      - name: Build with Maven
        run: mvn -B clean package -DskipTests=false

  deploy:
    needs: build-and-test
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'
    steps:
      - uses: actions/checkout@v4
      - name: Upload artifact
        uses: actions/upload-artifact@v4
        with:
          name: app-jar
          path: target/*.jar
      - name: Deploy to server
        uses: appleboy/ssh-action@v0.1.7
        with:
          host: ${{ secrets.SERVER_HOST }}
          username: ${{ secrets.SERVER_USER }}
          key: ${{ secrets.SERVER_SSH_KEY }}
          port: ${{ secrets.SERVER_SSH_PORT }}
          script: |
            set -e
            mkdir -p ~/deploys/ec-app
            rm -f ~/deploys/ec-app/app.jar
            scp -P ${{ secrets.SERVER_SSH_PORT }} ./target/*.jar ${{ secrets.SERVER_USER }}@${{ secrets.SERVER_HOST }}:~/deploys/ec-app/app.jar
            # restart systemd service
            sudo systemctl daemon-reload
            sudo systemctl restart ec-app.service

注：上記では appleboy/ssh-action を例示しています。実際にはセキュリティポリシーに従い、秘密鍵は GitHub Secrets に登録してください。

4.3 Ubuntu 側デプロイ手順（受け取り側）

# サーバー側（Ubuntu）での準備（初回のみ）
# 1. デプロイ用ユーザー作成（例: deploy）
sudo adduser deploy
sudo usermod -aG sudo deploy

# 2. systemd サービス定義
sudo tee /etc/systemd/system/ec-app.service > /dev/null <<'EOF'
[Unit]
Description=EC App
After=network.target

[Service]
User=deploy
WorkingDirectory=/home/deploy/deploys/ec-app
ExecStart=/usr/bin/java -jar /home/deploy/deploys/ec-app/app.jar --spring.profiles.active=prod
SuccessExitStatus=143
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable ec-app.service
sudo systemctl start ec-app.service
セキュリティ：SSH 鍵はパスフレーズ付きで管理し、GitHub Secrets に登録。サーバーのファイアウォール（ufw）で必要ポートのみ開放。

5 運用・品質・セキュリティ

5.1 セキュリティ対策（必須）

  • パスワードは必ず bcrypt でハッシュ化。
  • JWT の署名鍵は環境変数で管理（秘密鍵はリポジトリに置かない）。
  • SQL は ORM（JPA）を使い、プレースホルダを利用。
  • HTTPS：本番では Nginx で TLS 終端（Let's Encrypt）を設定。
  • ログに機密情報（パスワード、カード番号）は絶対に出力しない。

5.2 ロギング・監視

  • ログ：Logback を利用し、INFO/ERROR を分けて出力。
  • 監視：Prometheus + Grafana（将来的に）または簡易に systemd の再起動監視。

5.3 バックアップ

PostgreSQL の定期バックアップ（pg_dump）を cron で実行し、外部ストレージへ保存。

9 推奨学習順（若手エンジニア向け、参考書スタイルの短説明）

  1. Java と Spring Boot の基礎：DI、Controller/Service/Repository の役割を理解する。
  2. REST API 設計：ステータスコード、エラーフォーマット、ページネーション設計を学ぶ。
  3. RDB と SQL（PostgreSQL）：正規化、インデックス、トランザクションの基礎。
  4. セキュリティ：パスワードハッシュ、JWT、環境変数管理。
  5. CI/CD（GitHub Actions）：ビルド→テスト→デプロイの自動化。
  6. 運用：systemd、ログ、バックアップ、監視の基礎。

各項目は実装しながら学ぶのが最短です。まずは「小さく動く機能」を作ってテストと CI を回す習慣をつけてください。
